# 要件定義: 天秤.ai クローン（ローカル版）

## 1. プロジェクト概要

天秤.ai（https://tenbin.ai/workspace/welcome）と同等のシステムをローカルで構築する。
1つの入力欄に入力したプロンプトを複数のAIモデルに同時に投稿し、それぞれの回答を横並びで比較表示するWebアプリケーション。

### 前提条件

- ローカル専用（デプロイ不要）
- DB不要（永続化不要、ブラウザリロードで会話はリセット）
- APIキーは `.env.local` で管理（gitignore対象）
- レスポンシブ対応不要
- ダークモード不要

## 2. 対応AIサービス・モデル

4つのAIサービスに対応する。各サービスから1モデルを使用する。

| サービス | デフォルトモデル | SDK (npm) |
|---|---|---|
| OpenAI | gpt-4.1 | `openai` |
| Anthropic Claude | claude-sonnet-4-6 | `@anthropic-ai/sdk` |
| Google Gemini | gemini-2.5-flash | `@google/genai` |
| Perplexity | sonar-pro | OpenAI互換API |

### モデル設定

- 設定ファイル形式: TypeScript (`config/models.ts`)
- 設定ファイルで以下を定義:
  - 各サービスのモデルID
  - 調整可能なパラメータ（temperature等）
  - コメントとして各サービスのモデル比較サイトURLを記載
- UIからの動的なパラメータ変更は不要（設定ファイルを直接編集して変更）

## 4. 機能要件

### 4.1 プロンプト入力・送信

- テキスト入力欄にプロンプトを入力
- 送信先のモデルをチェックボックスで選択可能（デフォルトは4つすべて選択）
- 送信ボタン押下で、選択されたモデルに並列でリクエストを送信
- 送信中は「考え中」であることがわかるUI表示（ローディングインジケータ等）
- ストリーミング表示は不要。応答が返ったらまとめて表示する

### 4.2 会話の継続

- 最初の送信後、続けて追加のプロンプトを入力できる（チャット形式）
- 2回目以降の送信で、送信先モデルを変更できる
  - 例: 1回目はA,B,C,Dに送信 → 2回目はA,Bのみ → 3回目はAのみ
- 各モデルは自分に送信された会話履歴のみを保持する
- 会話コンテキスト（過去のやり取り）はAPIリクエストに含めて送信する

### 4.3 回答表示

- 各モデルの回答を横並び4カラムで表示
- 各カラムに表示する情報:
  - モデル名（サービス名）
  - トークン数（使用トークン）
  - 回答本文
- Markdownレンダリング対応（コードブロックのシンタックスハイライト含む）
- 回答のコピーボタン（クリップボードにコピー）

### 4.4 カラムの拡大表示

- カラムのヘッダー（タイトル）部分をクリックすると全幅に拡大表示
- 再度ヘッダーをクリックすると元の4カラム表示に戻る
- メッセージ本文部分のクリックでは拡大/縮小しない
- 長い回答を読みやすくするための機能

### 4.5 インターネット検索（Web Search）

- 全モデルで常にインターネット検索を有効化する（ユーザー設定によるON/OFFは設けない）
- 各サービスの検索機能を以下のとおり利用する:

| サービス | 検索の利用方法 |
|---|---|
| OpenAI | Responses API + `web_search` ツール |
| Anthropic Claude | Messages API + `web_search` ツール |
| Google Gemini | `generateContent` + `googleSearch` ツール |
| Perplexity | モデル本体が検索ベース（追加設定不要） |

- 検索結果として参照したURL（citations）が取得できる場合は、回答に併記する
- 検索の有無による応答時間のばらつきは許容する

### 4.6 エラーハンドリング

- APIキー未設定、レート制限、ネットワークエラー等の場合
- 該当モデルのカラム内にエラーメッセージを表示
- 他のモデルの表示には影響しない（個別にエラー処理）

### 4.7 チャットログ保存

- 各モデルの会話ログを `log/` フォルダにテキストファイルとして自動保存
- 1モデル = 1ファイル（セッション中に4モデルを使用した場合は4ファイル作成）
- ファイル名形式: `YYYY-MM-DD_HHmm_サービス名_モデルID.txt`
  - 例: `2026-04-02_1030_OpenAI_gpt-4.1.txt`
- 回答が返るたびにファイルに追記（ブラウザを閉じてもログが残る）
- ファイル内容:
  - Markdownの素の状態（レンダリング前）で出力
  - 発話者の区別: `[User]` / `[Assistant]`
  - 各発話の日時を記載
  - 会話の区切り: `--------------------------------------------------------`
- リセット時に新しいセッションとなり、次回送信時から新しいファイルが作成される
- `log/` フォルダはgitignore対象

### 4.8 リセット

- リセットボタンで会話をすべてクリア
- 全モデルの回答表示をクリアし、初期状態に戻る

## 5. 設定ファイル仕様

### 配置場所

`config/models.ts`

### 設定内容

```typescript
// 設定ファイルのイメージ（最終的な型定義は設計フェーズで決定）
{
  id: string;           // 内部識別子
  name: string;         // 表示名
  provider: string;     // サービス名
  modelId: string;      // APIに送信するモデルID
  enabled: boolean;     // デフォルトで有効か
  parameters: {
    temperature?: number;
    maxTokens?: number;
    // その他モデル固有のパラメータ
  };
}
```

### コメント要件

設定ファイル内に、各サービスのモデル比較・一覧ページのURLをコメントとして記載する。
モデル変更時の参考情報とする。

## 6. APIキー管理

### `.env.local` に格納するキー

```
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
GOOGLE_API_KEY=AI...
PERPLEXITY_API_KEY=pplx-...
```

- `.env.local` はgitignore対象（`.gitignore`で`.env.*`が除外済み）
- `.env.example` をリポジトリに含め、必要なキー名を記載する

## 7. 技術スタック

| 項目 | 技術 |
|---|---|
| フレームワーク | Next.js (App Router) |
| スタイリング | CSS Modules |
| Markdownレンダリング | react-markdown + シンタックスハイライト |
| テスト | Jest + Playwright |
| タスク管理 | Beads |

## 8. 非機能要件

- ローカル環境（localhost）でのみ動作すれば良い
- 同時4モデルへの並列リクエストを処理できること
- 各モデルの応答待ち中に他のモデルの結果が先に表示されても良い（到着順表示）
